# Feature: Bilinguale App (DE/EN) mit next-intl

## Ziel
Die App soll vollständig auf Deutsch und Englisch verfügbar sein. User wählen ihre Sprache in Settings. Die Wahl wird in Supabase (`profiles.language`) gespeichert und beim nächsten Login wiederhergestellt.

## Strategie (wichtig — nicht abweichen)

**NICHT** alle Strings in Supabase speichern — das wäre ein extra DB-Request bei jedem Page-Load.

Stattdessen:
- UI-Strings → zwei JSON-Dateien im Repo (`messages/de.json` + `messages/en.json`)
- User-Sprachpräferenz → ein Feld `language text DEFAULT 'de'` in `profiles`-Tabelle
- Library: `next-intl` (Standard für Next.js App Router i18n)

---

## Schritt 1: Supabase Migration

```sql
ALTER TABLE profiles
  ADD COLUMN IF NOT EXISTS language text DEFAULT 'de';
```

Führe diese Migration in Supabase aus (SQL Editor).

## Schritt 4: next-intl Setup für App Router

Erstelle `i18n.ts` im Root:
```ts
import { getRequestConfig } from 'next-intl/server';

export default getRequestConfig(async () => {
  // Default DE — wird später durch User-Präferenz aus Supabase ersetzt
  const locale = 'de';
  return {
    locale,
    messages: (await import(`./messages/${locale}.json`)).default
  };
});
```

Update `next.config.js` (oder `next.config.ts`):
```js
const createNextIntlPlugin = require('next-intl/plugin');
const withNextIntl = createNextIntlPlugin('./i18n.ts');

module.exports = withNextIntl({
  // ...deine bestehende Config
});
```

## Schritt 5: Language Context Provider

Erstelle `components/LanguageProvider.tsx`:
```tsx
'use client';
import { createContext, useContext, useEffect, useState } from 'react';
import { createClientComponentClient } from '@supabase/auth-helpers-nextjs';

type Language = 'de' | 'en';

const LanguageContext = createContext<{
  language: Language;
  setLanguage: (lang: Language) => void;
}>({ language: 'de', setLanguage: () => {} });

export function LanguageProvider({ children }: { children: React.ReactNode }) {
  const [language, setLanguageState] = useState<Language>('de');
  const supabase = createClientComponentClient();

  useEffect(() => {
    // Lade Sprache aus Supabase beim Start
    supabase.auth.getUser().then(({ data: { user } }) => {
      if (!user) return;
      supabase
        .from('profiles')
        .select('language')
        .eq('id', user.id)
        .single()
        .then(({ data }) => {
          if (data?.language) setLanguageState(data.language as Language);
        });
    });
  }, []);

  const setLanguage = async (lang: Language) => {
    setLanguageState(lang);
    const { data: { user } } = await supabase.auth.getUser();
    if (user) {
      await supabase.from('profiles').update({ language: lang }).eq('id', user.id);
    }
  };

  return (
    <LanguageContext.Provider value={{ language, setLanguage }}>
      {children}
    </LanguageContext.Provider>
  );
}

export const useLanguage = () => useContext(LanguageContext);
```

Wrap `app/layout.tsx` mit `<LanguageProvider>`.

## Schritt 6: Sprach-Toggle in Settings

In `app/(protected)/settings/page.tsx` einen Language-Selector hinzufügen:

```tsx
import { useLanguage } from '@/components/LanguageProvider';

// Inside component:
const { language, setLanguage } = useLanguage();

// UI:
<div>
  <h3>Sprache / Language</h3>
  <div style={{ display: 'flex', gap: 8 }}>
    <button
      onClick={() => setLanguage('de')}
      style={{
        background: language === 'de' ? '#4F6EF7' : 'transparent',
        color: language === 'de' ? 'white' : '#4F6EF7',
        border: '1px solid #4F6EF7',
        borderRadius: 8,
        padding: '8px 16px'
      }}
    >
      🇩🇪 Deutsch
    </button>
    <button
      onClick={() => setLanguage('en')}
      style={{
        background: language === 'en' ? '#4F6EF7' : 'transparent',
        color: language === 'en' ? 'white' : '#4F6EF7',
        border: '1px solid #4F6EF7',
        borderRadius: 8,
        padding: '8px 16px'
      }}
    >
      🇬🇧 English
    </button>
  </div>
</div>
```

## Schritt 7: Strings in Komponenten ersetzen (Priorität: /log zuerst)

In jeder Komponente:
```tsx
import { useTranslations } from 'next-intl';

// Inside component:
const t = useTranslations('log');

// Verwendung:
<label>{t('carbs')}</label>  // → "Kohlenhydrate" oder "Carbohydrates"
```

Fange mit `app/(protected)/log/page.tsx` an (größte Wirkung), dann `/settings`, dann `/history`, dann Layout-Nav.

---

## Wichtige Hinweise

- `tsc --noEmit` nach jedem Schritt ausführen
- Erst Schritt 1-4 (Setup) commiten, dann Schritt 5-7 schrittweise
- Fehlende Strings in den JSON-Dateien werden von next-intl als Fehler geloggt — gut so, so findet man fehlende Übersetzungen sofort
- Fallback-Locale ist `de` — wenn ein String fehlt, zeigt next-intl den deutschen Text
