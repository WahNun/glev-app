## 1. IC-Ratio Two-Value System
**Was erwartet wird:**
- Tabelle users/profile hat Spalten: `icr_g_per_unit` (User-Wert), `icr_g_per_unit_engine` (Engine-Wert), `engine_icr_auto_apply` (boolean, default false)
- Bolus-Kalkulator liest standardmäßig `icr_g_per_unit`
- Wenn `engine_icr_auto_apply = true`, wird `icr_g_per_unit_engine` genutzt
- Engine schreibt NUR in `icr_g_per_unit_engine`, nie in `icr_g_per_unit`
- UI zeigt beide Werte getrennt an (User-Wert + Engine-Vorschlag)
- Toggle in Settings vorhanden

**Prüfe:** Supabase-Schema, Bolus-Kalkulator-Logik, Settings-UI
