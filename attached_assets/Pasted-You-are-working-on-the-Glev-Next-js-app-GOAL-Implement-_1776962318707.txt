# PART 1 — OUTCOME ENGINE

## RULE

Outcome must NOT be calculated immediately.

---

## REQUIRED DATA (FINAL)

- bgBefore
- bg1h
- bg2h

---

## INTERMEDIATE STATE

After 1h:

→ compute provisional outcome

States:
- "pending"
- "provisional"
- "final"

---

## LABELS

- pending → "Awaiting glucose data"
- provisional → "Preliminary result (updates after 2h)"
- final → actual outcome

---

## METRICS

- delta1 = bg1h - bgBefore
- delta2 = bg2h - bgBefore
- speed1 = delta1 / 60min
- speed2 = delta2 / 120min

---

## PROVISIONAL LOGIC

Based on:

- delta1
- speed1

---

## FINAL LOGIC

Based on:

- delta1
- delta2
- speed1
- speed2
- spike pattern

# PART 6 — ADAPTIVE CARB RATIO ENGINE

## CORE

effectiveCarbRatio = carbs / insulin

---

## FILTER

Only use:

- outcomeState === "final"
- carbs > 0
- insulin > 0

---

## WEIGHTING

- GOOD → 1.0
- SPIKE → 0.7
- UNDERDOSE → 0.3
- OVERDOSE → 0.3

---

## OUTPUT

adaptiveICR = {
  global,
  morning?,
  afternoon?,
  evening?
}

---

# PART 7 — INSULIN RECOMMENDATION ENGINE

## INPUT

- carbs
- currentBG
- targetBG (100)
- adaptiveICR
- correctionFactor

---

## CALCULATION

carbDose = carbs / adaptiveICR  
correctionDose = (currentBG - targetBG) / correctionFactor  
totalDose = carbDose + correctionDose  

---

## SAFETY

- carbs = 0 allowed
- BG < 80 → block
- round units

---

## OUTPUT

{
  recommendedUnits,
  breakdown,
  reasoning,
  confidence
}

---

# PART 8 — PATTERN DETECTION ENGINE (NEW CORE)

## GOAL

Detect recurring dosing mistakes across recent meals.

---

## WINDOW

Use rolling window:

- last 10–20 meals
- only outcomeState === "final"

---

## ANALYSIS

Count:

- GOOD
- UNDERDOSE
- OVERDOSE
- SPIKE

---

## THRESHOLDS

IF:

- >50% OVERDOSE → pattern = "overdosing"
- >50% UNDERDOSE → pattern = "underdosing"

---

## OUTPUT

pattern = {
  type: "overdosing" | "underdosing" | "balanced",
  confidence: low | medium | high,
  sampleSize: number
}

---

## UI

Display:

- "Frequent Over-dosing"
- "Consistent Under-dosing"

+ short explanation

---

# PART 9 — ADJUSTMENT ENGINE (AUTO-LEARNING)

## GOAL

Adapt ICR + correctionFactor gradually based on patterns.

---

## RULE

NEVER adjust on single meal.

ONLY adjust when:

- pattern confidence = medium or high
- sampleSize >= 5

---

## ADJUSTMENT LOGIC

### IF overdosing

→ insulin too strong

- increase ICR (less insulin per carb)
- increase correctionFactor

---

### IF underdosing

→ insulin too weak

- decrease ICR
- decrease correctionFactor

---

## ADJUSTMENT STEP

- max change per update: ±5%
- smooth incremental updates

---

## EXAMPLE

ICR:
1:10 → 1:10.5 → 1:11

---

## STORAGE

Store:

adaptiveSettings = {
  icr,
  correctionFactor,
  lastUpdated,
  adjustmentHistory[]
}

---

## UX RULE

DO NOT silently change.

Always show:

"Based on recent meals, your carb ratio was adjusted from 1:10 → 1:11"

---

## MODE

Start in:

→ suggestion mode (user confirms)

Later optional:

→ auto-adjust mode
