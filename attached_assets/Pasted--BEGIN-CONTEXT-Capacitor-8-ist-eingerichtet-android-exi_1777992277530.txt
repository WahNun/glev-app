STEP 3 — Signing-Konfiguration hinzufügen
In android/app/build.gradle einen signingConfigs-Block hinzufügen und im release-BuildType referenzieren:
gradleandroid {
    signingConfigs {
        release {
            storeFile file(System.getenv("KEYSTORE_PATH") ?: "release.keystore")
            storePassword System.getenv("KEYSTORE_PASSWORD") ?: ""
            keyAlias System.getenv("KEY_ALIAS") ?: "glev"
            keyPassword System.getenv("KEY_PASSWORD") ?: ""
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}
Die Werte kommen aus Umgebungsvariablen — der Keystore selbst wird lokal auf dem Mac generiert, nicht in Replit.

STEP 4 — Android-Scripts in package.json ergänzen
Füge zur scripts-Sektion in package.json hinzu:
json"android:sync": "npx cap sync android",
"android:open": "npx cap open android",
"android:build:debug": "npx cap sync android && cd android && ./gradlew assembleDebug",
"android:version": "node scripts/bump-android-version.mjs show"
Erstelle scripts/bump-android-version.mjs (analog zu scripts/bump-ios-version.mjs):
javascriptimport { readFileSync, writeFileSync } from 'fs'

const buildGradlePath = 'android/app/build.gradle'
const content = readFileSync(buildGradlePath, 'utf8')

const versionCodeMatch = content.match(/versionCode (\d+)/)
const versionNameMatch = content.match(/versionName "([^"]+)"/)

const versionCode = versionCodeMatch ? parseInt(versionCodeMatch[1]) : 0
const versionName = versionNameMatch ? versionNameMatch[1] : '0.0.0'

const arg = process.argv[2]

if (arg === 'show') {
  console.log(`Android versionCode: ${versionCode}, versionName: ${versionName}`)
} else if (arg === 'bump') {
  const newVersionCode = versionCode + 1
  const newContent = content
    .replace(/versionCode \d+/, `versionCode ${newVersionCode}`)
  writeFileSync(buildGradlePath, newContent)
  console.log(`versionCode: ${versionCode} → ${newVersionCode}`)
}

STEP 5 — Keystore-Anleitung als README
Erstelle android/SIGNING_SETUP.md:
markdown# Android Signing Setup

## Keystore einmalig generieren (lokal auf dem Mac)

\`\`\`bash
keytool -genkey -v \
  -keystore ~/glev-release.keystore \
  -alias glev \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000
\`\`\`

Beim Ausfüllen: Name, Organisation (Glev), Land (PT), Passwort merken.

⚠️ Den Keystore NIEMALS ins Git committen. Er bleibt lokal auf deinem Mac.
Backup: iCloud oder Passwort-Manager.

## Umgebungsvariablen setzen (lokal, vor dem Build)

\`\`\`bash
export KEYSTORE_PATH=~/glev-release.keystore
export KEYSTORE_PASSWORD=<dein_passwort>
export KEY_ALIAS=glev
export KEY_PASSWORD=<dein_passwort>
\`\`\`

## AAB bauen für Play Store

\`\`\`bash
npm run android:sync
cd android
./gradlew bundleRelease
\`\`\`

AAB liegt danach unter: android/app/build/outputs/bundle/release/app-release.aab

## In Play Console hochladen

1. play.google.com/console → Glev → Testing → Internal testing
2. Create new release → AAB hochladen
3. Release notes hinzufügen → Save → Review → Start rollout
\`\`\`
