🌾 17. ADD FIBER TRACKING (IMPORTANT)
--------------------------------------------------

Extend the data model to include fiber.

Update Entry type:

Add:
fiber?: number;

--------------------------------------------------
📊 DATA MODEL UPDATE
--------------------------------------------------

Each entry must now support:

- carbs (total carbs)
- fiber (grams)

IMPORTANT:
Fiber must be tracked separately from carbs.

--------------------------------------------------
🧠 CALCULATION UPDATE (NET CARBS)
--------------------------------------------------

Introduce:

net_carbs = carbs - fiber

Rules:
- if fiber is missing → assume 0
- net_carbs cannot be negative

Use net_carbs for:
- insulin recommendation
- meal classification

--------------------------------------------------
🔥 RECOMMENDATION ENGINE UPDATE
--------------------------------------------------

Replace:

units = carbs / ratio

WITH:

units = net_carbs / ratio

Reason:
Fiber slows glucose absorption and reduces effective carb load.

--------------------------------------------------
🧠 CLASSIFIER UPDATE
--------------------------------------------------

Update classification logic:

If fiber > 10g:
→ reduce likelihood of FAST_CARBS classification

If fiber > 15g:
→ bias toward BALANCED or HIGH_FAT

Example:
High carb + high fiber ≠ FAST_CARBS
