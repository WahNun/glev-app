Bug 1: Eingeladene User landen nach Klick auf "Accept Invitation" auf der Homepage (/) statt auf einem Passwort-Setup-Screen.

Bug 2: Manuell angelegte User erhalten keine Email oder sehen "email not confirmed".

## SCHRITT 2 — Sofort-Fix für bereits angelegte User

## SCHRITT 3 — Auth Callback Route

Erstelle app/auth/confirm/route.ts:
- Liest token_hash und type aus URL-Parametern
- supabase.auth.verifyOtp({ token_hash, type })
- Bei type 'invite' oder 'signup' → redirect zu /welcome/setup
- Bei Fehler → redirect zu /auth/error?message=...

---

## SCHRITT 4 — Passwort-Setup Seite

Erstelle app/welcome/setup/page.tsx:
- 'use client'
- Formular: Passwort + Passwort bestätigen (min 8 Zeichen)
- supabase.auth.updateUser({ password })
- Nach Erfolg: redirect zu /dashboard
- Nur Tailwind, keine neuen Libraries

---

## SCHRITT 5 — Admin Invite Route

Erstelle app/api/admin/invite/route.ts:
- POST mit { email }
- supabaseAdmin.auth.admin.inviteUserByEmail(email, { redirectTo: process.env.NEXT_PUBLIC_SITE_URL + '/auth/confirm' })
- Gibt { success: true, userId } zurück

Erstelle lib/supabase/admin.ts falls nicht vorhanden:
- createClient(url, SUPABASE_SERVICE_ROLE_KEY, { auth: { autoRefreshToken: false, persistSession: false } })

Neue Env Var: NEXT_PUBLIC_SITE_URL=https://glev.app
