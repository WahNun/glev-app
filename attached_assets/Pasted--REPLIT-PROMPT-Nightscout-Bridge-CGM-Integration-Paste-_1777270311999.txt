## CONTEXT

Glev already integrates FreeStyle Libre 2 via LibreLinkUp. We now want to add a second CGM source: Nightscout. Nightscout is an open-source CGM data platform used by Dexcom, Libre, and (soon) Accu-Chek SmartGuide users. Adding this one integration makes Glev compatible with virtually all major CGMs without touching device-specific APIs.

---

## GOAL

User enters their Nightscout URL + API Secret in Glev's settings. Glev polls their Nightscout instance every 5 minutes, stores readings in Supabase, and displays them on the dashboard exactly like LibreLinkUp glucose data.

---

## STEP 1 — Add Nightscout fields to user settings

Find where app settings/profile are stored (likely `lib/supabase/profiles` or similar, and a settings page under `app/(protected)/settings`).

Add two new fields to the user profile (Supabase `profiles` table or equivalent):
- `nightscout_url` (text, nullable) — e.g. `https://mynightscout.fly.dev`
- `nightscout_token` (text, nullable) — the Nightscout API secret or read token

In the settings UI, add a new section titled "Nightscout" with:
- Input: "Nightscout URL" (placeholder: `https://mynightscout.fly.dev`)
- Input: "API Secret / Token" (type=password)
- Save button
- Small helper text below: "Nightscout ist eine Open-Source-CGM-Plattform. Kompatibel mit Dexcom, FreeStyle Libre und anderen."

## STEP 2 — Create the Nightscout polling API route

Create `app/api/cgm/nightscout/sync/route.ts`.

```ts
import { NextRequest, NextResponse } from 'next/server'
import { createClient } from '@/lib/supabase/server'

export async function POST(req: NextRequest) {
  const supabase = createClient()

  // Get authenticated user
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  // Get Nightscout credentials from profile
  const { data: profile } = await supabase
    .from('profiles')
    .select('nightscout_url, nightscout_token')
    .eq('id', user.id)
    .single()

  if (!profile?.nightscout_url) {
    return NextResponse.json({ error: 'No Nightscout URL configured' }, { status: 400 })
  }

  const baseUrl = profile.nightscout_url.replace(/\/$/, '')
  const token = profile.nightscout_token

  // Fetch last 12 readings (1 hour at 5-min intervals)
  const url = `${baseUrl}/api/v1/entries.json?count=12${token ? `&token=${token}` : ''}`

  let entries
  try {
    const res = await fetch(url, { headers: { Accept: 'application/json' } })
    if (!res.ok) throw new Error(`Nightscout returned ${res.status}`)
    entries = await res.json()
  } catch (e: any) {
    return NextResponse.json({ error: `Nightscout fetch failed: ${e.message}` }, { status: 502 })
  }

  // Map to Glev glucose readings format
  // Nightscout entry: { sgv: number, date: number (unix ms), direction: string }
  const readings = entries
    .filter((e: any) => e.sgv && e.date)
    .map((e: any) => ({
      user_id: user.id,
      value_mgdl: e.sgv,
      recorded_at: new Date(e.date).toISOString(),
      source: 'nightscout',
      trend: e.direction ?? null,
    }))

  if (readings.length === 0) {
    return NextResponse.json({ synced: 0 })
  }

  // Upsert into glucose_readings (or whatever the existing table is called)
  // Use recorded_at + user_id as the conflict target to avoid duplicates
  const { error } = await supabase
    .from('glucose_readings')
    .upsert(readings, { onConflict: 'user_id,recorded_at' })

  if (error) {
    return NextResponse.json({ error: error.message }, { status: 500 })
  }

  return NextResponse.json({ synced: readings.length })
}
```

**IMPORTANT:** Check what the actual glucose readings table is called in this codebase before using `glucose_readings`. Search for where LibreLinkUp data is stored and use the same table name + column names.
