Build a modern, production-ready web app for a Type 1 Diabetes insulin tracking and decision-support system.

The app must be deployable on Vercel and structured for Git-based development.

Stack requirements:
- Next.js (App Router)
- React
- Tailwind CSS
- Serverless API routes (Vercel)
- Google Sheets as database (initially)

Focus:
- ultra-fast logging
- batch import (copy/paste + CSV)
- insulin recommendation engine
- clean UX
- simple, scalable architecture

--------------------------------------------------
🧠 CORE SYSTEM LOGIC
--------------------------------------------------

The system evaluates insulin effectiveness using:

- Glucose before meal
- Glucose after meal
- Time difference
- Carbohydrate intake
- Insulin (bolus units)

Derived metrics:
- Delta
- Duration
- Speed (mg/dL per hour)

Classification:
- OVERDOSE
- UNDERDOSE
- GOOD
- CHECK_CONTEXT

Meal types:
- FAST_CARBS
- HIGH_FAT
- HIGH_PROTEIN
- BALANCED

--------------------------------------------------
🏗️ ARCHITECTURE
--------------------------------------------------

Use clean separation:

1. UI (React + Tailwind)
2. API Layer (Next.js API routes)
3. Data Layer (Google Sheets API)

Structure:

/app
  /dashboard
  /log
  /import
  /insights
  /recommendation

/api
  /log-meal
  /import-batch
  /upload-csv
  /get-entries

/lib
  /calculation (delta, speed, evaluation)
  /recommendation
  /parser (CSV + paste parser)
  /google-sheets

--------------------------------------------------
📥 INPUT SYSTEM (CRITICAL)
--------------------------------------------------

Support 3 input methods:

1. QUICK ENTRY
- minimal fields
- <10 seconds
- mobile-first

2. COPY-PASTE IMPORT
- user pastes tabular data
- system parses automatically
- handles tab-separated input

3. CSV UPLOAD
- drag & drop
- supports comma or semicolon
- preview + confirm import

--------------------------------------------------
🧠 IMPORT FLOW
--------------------------------------------------

1. User pastes or uploads
2. System parses data
3. Shows preview table
4. User confirms
5. Data sent to API
6. Stored in RAW_IMPORT (Google Sheet)
7. CLEAN_DATA auto-processed
