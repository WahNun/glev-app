Add iOS-style drag-and-drop reordering of cards on the Dashboard and Insights screens.

Interaction pattern (matching iOS home screen):
- User long-presses any card (~500ms hold) → enters "edit mode"
- All cards start a subtle wiggle animation
- User can drag any card to a new position; other cards shift to make space
- Tapping empty blank space (outside any card) exits edit mode and persists the new order
- Order persists per user across devices/sessions

=== DEPENDENCIES ===

STEP 1 — Install:
  npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities

=== PERSISTENCE LAYER ===

STEP 2 — Add Supabase schema for user preferences. Run this SQL in Supabase:

create table if not exists public.user_preferences (
  user_id uuid primary key references auth.users(id) on delete cascade,
  dashboard_card_order jsonb default '[]'::jsonb,
  insights_card_order jsonb default '[]'::jsonb,
  updated_at timestamptz default now()
);

alter table public.user_preferences enable row level security;

create policy "Users can manage own preferences"
  on public.user_preferences
  for all
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);

STEP 3 — Create two tiny API routes:
  - GET  /api/preferences — returns { dashboard_card_order, insights_card_order } for the current user (cookie auth via the existing authenticate() helper in app/api/cgm/_helpers.ts)
  - POST /api/preferences — accepts { dashboard_card_order?: string[], insights_card_order?: string[] } and upserts to user_preferences

=== REUSABLE DnD COMPONENT ===

STEP 4 — Create components/SortableCardGrid.tsx — a generic wrapper that:
  - Takes children cards + an `orderKey` prop ("dashboard" or "insights") + an `onOrderChange` callback
  - Detects long-press on any card (500ms hold) to enter edit mode
  - While in edit mode, applies the wiggle CSS class to all cards
  - Uses @dnd-kit/sortable's SortableContext + useSortable on each card
  - Exits edit mode when user taps outside any card (document-level click listener while active, cancelling if the click hits a card)
  - On exit: calls onOrderChange with the new ID array

STEP 5 — Wiggle animation in globals.css:

@keyframes wiggle {
  0%, 100% { transform: rotate(-0.8deg); }
  50% { transform: rotate(0.8deg); }
}
.wiggle {
  animation: wiggle 0.2s ease-in-out infinite;
  transform-origin: center;
}
/* Disable wiggle for users who prefer reduced motion */
@media (prefers-reduced-motion: reduce) {
  .wiggle { animation: none; }
}

STEP 6 — Add a small "x" delete icon overlay on each card while in edit mode (top-right corner). For now it does nothing (or just hides the card from view locally — we'll wire real deletion later). Goal is just the iOS look-and-feel.
