# Fix Report — Playwright-Tests für screen-orientation API-Aufrufe in LandscapeGlucoseOverlay
**Task GID:** 1068
**Date:** 2026-06-03

## Was wurde gebaut

Automatisierte Playwright-Tests, die verifizieren dass `LandscapeGlucoseOverlay` die `@capacitor/screen-orientation`-API korrekt aufruft (Task #1068).

Neuer `describe`-Block 4 in `tests/e2e/landscape-glucose-overlay.spec.ts`:

- **Test A** — `unlock()` wird beim Component-Mount aufgerufen (Capacitor-Native-Kontext)
- **Test B** — `lock({ orientation: "portrait" })` wird beim landscape→portrait-Übergang aufgerufen
- **Test C** — In einem normalen Browser (`isNativePlatform = false`) werden keine Orientation-API-Calls gemacht

## Technischer Ansatz: Capacitor Bridge Mock via `addInitScript`

Das Kernproblem war, wie man `@capacitor/screen-orientation`-Aufrufe im Playwright-Kontext abfangen kann, ohne echte Capacitor-Infrastruktur zu benötigen.

**Lösung:** `page.addInitScript(nativeCapacitorScript)` setzt `window.Capacitor` *vor* Capacitor Core initialisiert:

1. **`PluginHeaders`** — Capacitor Core's `createCapacitor()` prüft `win.Capacitor ?? {}` und preserviert vorhandene Felder. `isNativePlatform()` gibt `true` zurück wenn `PluginHeaders` existiert. Das `methods`-Array mit `unlock` und `lock` teilt Capacitor mit, diese Methoden über `nativePromise` zu routen.

2. **`nativePromise` Spy** — `createCapacitor()` überschreibt `nativePromise` **nicht** (bestätigt durch Lektüre von `@capacitor/core/dist/index.cjs.js`). Alle `ScreenOrientation.unlock()` und `ScreenOrientation.lock()` Aufrufe laufen durch unseren Spy, der sie in `window.__screenOrientationCalls` aufzeichnet.

3. **Test C (Browser-Kontext)** — kein `PluginHeaders` → `isNativePlatform()` = false → `loadScreenOrientation()` in `LandscapeGlucoseOverlay.tsx` gibt `null` zurück → keine Orientation-API-Calls.
