ARCHITECTURE NOTES
------------------
- Zwei Signup-Pfade existieren jetzt parallel:
    (A) Payment-First: Stripe Checkout → Webhook → signUp (Beta, Pro)
    (B) Free-Trial:    signUp direkt → /api/auth/free-trial (kein Stripe)

- trial_end_at Semantik:
    NULL       = kein Trial (reguläre Free/Paid-User) — kein Modal
    != NULL    = Trial-User
    < NOW()    = abgelaufen — Modal erscheint wenn plan === "free"

- Paywall client-seitig (Modal in layout.tsx), KEIN Middleware-Block
  Grund: Middleware kann keinen Admin-Client-Query machen; SSR-Block würde
  Flash-of-Gate für Paid-User erzeugen. Siehe D-023.

- Kein eigener "trial"-Plan-Typ in computeEffectivePlan.
  trial_end_at + plan === "free" reicht für alle Prüfungen.

- apply-migration.mjs: schema_migrations in Produktion hat sha256 TEXT NOT NULL.
  Script befüllt es jetzt via crypto.createHash('sha256').update(sql).digest('hex').
  Für manuelle SQL-Editor-Anwendung immer SHA256 mitangeben:
    INSERT INTO schema_migrations (filename, sha256) VALUES ('...', '...') ON CONFLICT DO NOTHING;
