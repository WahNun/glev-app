# Task #576 — Add insulin brand fields to user settings

## Summary
Added `insulin_brand_bolus` and `insulin_brand_basal` text fields throughout the Glev app: DB migration, settings UI, IOB footer, insulin log pre-fill, and engine auto-log.

## Changes

### DB Migration
- `supabase/migrations/20260522_add_insulin_brands.sql`: Two nullable TEXT columns added to `user_settings` with a `char_length <= 40` CHECK constraint.

### lib/userSettings.ts
- `InsulinSettings` interface: two new optional fields `insulinBrandBolus` and `insulinBrandBasal`
- `getInsulinSettings()`: reads brand strings from localStorage (`glev_settings`)
- `fetchInsulinSettings()`: selects `insulin_brand_bolus` and `insulin_brand_basal` from DB
- `saveInsulinSettings()`: includes both brand columns in the upsert payload

### app/(protected)/settings/page.tsx
- `Settings` interface and `DEFAULTS` extended with both brand fields (empty string defaults)
- `SheetKey` union type: `"insulinBrandBolus"` and `"insulinBrandBasal"` added
- `fetchInsulinSettings` callback now merges brand values into state
- New `saveInsulinBrandsAction` callback
- Two new sheet entries in `sheetContent`
- Two new `SettingsRow` entries in the Insulin section (after insulinType row)

### components/IOBCard.tsx
- Reads `insulinBrandBolus` from `getInsulinSettings()` via `useMemo`
- Shows the brand name in the DIA footer when set, falls back to the generic "Rapid"/"Regular" label

### components/EngineLogTab.tsx
- Added `import { getInsulinSettings }` from `@/lib/userSettings`
- `InsulinForm.name` state lazy-initialised with the saved bolus brand
- `handleTypeChange` updates the name field with the appropriate brand when switching tabs (only if field is empty or matches the previous tab's brand)

### app/(protected)/engine/page.tsx
- Added `insertInsulinLog` to the import from `@/lib/insulin`
- Three bolus save paths (`handleWizardSave`, `handleSaveDirectBolus`, `handleSaveWithEagerBolus`) now fire-and-forget an `insulin_logs` entry with the bolus brand name when dose > 0

### messages/de.json + messages/en.json
- 11 new keys each: `row_insulin_brand_bolus`, `row_insulin_brand_basal`, `sheet_insulin_brand_bolus_title`, `sheet_insulin_brand_basal_title`, `insulin_brand_bolus_placeholder`, `insulin_brand_basal_placeholder`, `insulin_brand_body`, `insulin_brand_hint`, `subtitle_no_brand`

## Type Safety
`npx tsc --noEmit` passes with zero errors in source files (pre-existing test-file errors unchanged).
