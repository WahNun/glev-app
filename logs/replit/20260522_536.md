# Task #536 — Footer & Nav Audit + Playwright Tests

## Summary

Audited all footer and navigation elements for functional defects and wrote
comprehensive Playwright E2E tests covering 5 areas: Landing Footer, Mobile
Bottom Nav, Desktop Sidebar, Legal Page tabs/links, and Mobile Header.

## Audit Findings (No Code Fixes Required)

- **Landing Footer** (`components/landing/Footer.tsx`): Both Impressum and
  Datenschutz links correctly point to `/legal`. The legal page defaults to
  the DSE tab which contains the Impressum contact section at the top — this
  is intentional. The mailto link (`hello@glev.app`) is correct.

- **Legal Page** (`app/legal/page.tsx`): All external links already have
  `target="_blank"` and `rel="noopener"` (CNPD, ODR platform, glev.app).
  Tab switching uses correct ARIA attributes (`role="tab"`, `aria-selected`,
  `aria-controls`). Tab panels use `display: none/block` CSS for visibility.

- **Mobile Header** (`components/Layout.tsx`): The "+" QuickAddMenu was
  removed from the header in the 2026-05-17 UX revision. The FAB in the
  bottom nav is now the sole quick-add entry point. The logo button correctly
  opens the AccountSheet (BottomSheet with `role="dialog"`).

- **Mobile Bottom Nav**: All 5 slots (Dashboard, Einträge, FAB, Insights,
  Einstell.) render correctly. FAB short-press routes to `/engine`.

- **Desktop Sidebar**: All 5 nav items present. Sign-Out calls `signOut()`
  then `router.push("/login")`.
