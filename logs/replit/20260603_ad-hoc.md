## Was wurde gebaut

## Root Cause der robotischen Stimme

Direkttest gegen Mistral API ergab: `speed`-Feld im Request-Body wird von voxtral-mini-tts-2603 **nicht ignoriert, sondern mit HTTP 422 abgelehnt** ("Extra inputs are not permitted"). Das verursachte in unserer Route einen 502, der App-seitig Fallback auf Web Speech API auslöste — Web Speech klingt robotisch. Fix: `speed` aus dem Mistral-Body entfernt. Bestätigt: ohne `speed` → HTTP 200 + `audio_data`.
