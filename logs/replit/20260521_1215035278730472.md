# IOB-Visualisierung: Dashboard-Gauge + Entries-Sparkline + i18n

**Task GID:** 1215035278730472  
**Datum:** 2026-05-21

## Was wurde gebaut

### `components/IOBCard.tsx` (neu)
- SVG-Kreis-Gauge (Radius 44, strokeWidth 9, Circumference ≈ 276px)
- Farbcodierung: grün < 1 IE · amber 1–3 IE · orange > 3 IE
- Zentrierter Wert (Mono-Font) + IE-Label als Overlay im Ring
- Status-Chip: „Abgebaut in ~X min" (farbig) oder „Vollständig abgebaut" (grau)
- Risiko-Hinweis-Zeile am Fuß (Kein / Moderat / Stacking-Risiko)
- Auto-Refresh alle 5 Minuten + `window.addEventListener("focus")`

### Dashboard (`app/(protected)/dashboard/page.tsx`)
- `insulinType`-State + `fetchInsulinType`-Call im Mount-Effect
- IOBCard als dritte Karte im Glukose-Cluster (nach Glucose-Live + Trend-Chart)
- Imports: `InsulinType` aus `lib/iob`, `IOBCard`, `fetchInsulinType` aus `lib/userSettings`

### Entries expanded view (`app/(protected)/entries/page.tsx`)
- Import: `calcSingleIOB`, `getDIAMinutes`, `BolusDose`, `InsulinType`, `fetchInsulinType`
- `insulinType`-State + einmaliger Fetch in eigenem `useEffect`
- Nach dem CLASSIFICATION-Block: SVG-Sparkline des parabolischen Wirkprofils
  - 61 Punkte entlang DIA-Achse, `polyline`-Elemente mit `clipPath`-Split
  - Grauer Overlay für den vergangenen Teil, farbige Linie für verbleibendes IOB
  - Gestrichelte Vertikallinie als „Jetzt"-Marker
  - Chip darunter: „Noch X.X IE wirksam" (farbig) oder „Vollständig abgebaut" (grau)
  - Nur sichtbar wenn `m.insulin_units > 0`

### i18n (`messages/de.json` + `messages/en.json`)
- Namespace `dashboard`: 6 neue Keys (`active_insulin`, `iob_cleared_in`, `iob_fully_cleared`, `iob_risk_low`, `iob_risk_moderate`, `iob_risk_high`)
- Namespace `entriesExpand`: 3 neue Keys (`iob_profile`, `iob_fully_cleared`, `iob_still_active`)

## Keine Breaking Changes
Keine Supabase-Migration, keine neuen API-Routen, kein Schema-Wechsel.  
Alle neuen Daten kommen aus dem bereits im Dashboard gefetchten `insulin`-Array (letzte 60 Min Bolus-Logs).
