# Task #696 — Phase 4: Voice Input via Voxtral STT in GlevAIChatSheet

## Was wurde gemacht

**Ziel:** Hold-to-talk Mikrofon-Button im `GlevAIChatSheet` — Spracheingabe via Mistral Voxtral STT, Transkript landet im Input-Feld zur manuellen Kontrolle.

### 1. Server-Route `app/api/transcribe/mistral/route.ts`
- POST-Route nimmt `multipart/form-data` mit `audio`-Blob
- Nutzt `getMistralClient()` → `mistral.audio.transcriptions.complete({ model: "voxtral-mini", file })`
- Auth-Gate via Supabase Session (kein unauthed Zugriff)
- Gibt `{ text: string }` zurück
- **Kein separater `MISTRAL_STT_KEY`** — `MISTRAL_API_KEY` deckt alle Mistral-Services ab (D-018)

### 2. Hook `hooks/useVoxtral.ts`
- `startListening()`: `navigator.mediaDevices.getUserMedia({ audio: true })` → `MediaRecorder` (audio/webm;codecs=opus Fallback)
- `stopListening()`: MediaRecorder stoppen → Blob zusammenbauen → POST an `/api/transcribe/mistral` → `onTranscript(text)` aufrufen
- SSR-safe (`typeof window === "undefined"` Guard)
- Läuft auf Web und Capacitor native (Capacitor bridged getUserMedia zu AVAudioSession/RECORD_AUDIO)
- **Kein `@capacitor/microphone`** — existiert nicht als npm-Paket; nicht nötig

### 3. `components/GlevAIChatSheet.tsx`
- Mic-Button (36×36, rund) links neben dem Text-Input eingefügt
- `onPointerDown` → `startListening()`, `onPointerUp` + `onPointerLeave` → `stopListening()`
- Pulsiert mit `glevBtnGlowFast`-Animation in `#4F6EF7` während Aufnahme; Ruhezustand `#21262d`
- Placeholder wechselt zu "Spreche …" während Aufnahme
- Input-Border nimmt Accent-Farbe an während Aufnahme
- STT-Fehler als kleiner roter Hinweis unter der Input-Row

### 4. `android/app/src/main/AndroidManifest.xml`
- `<uses-permission android:name="android.permission.RECORD_AUDIO" />` ergänzt
- iOS `NSMicrophoneUsageDescription` war bereits vorhanden — nicht verändert

## Entscheidungen

- **`MISTRAL_API_KEY` wiederverwendet** (nicht neues `MISTRAL_STT_KEY`): Mistral nutzt denselben API-Key für alle Services. Dokumentiert in D-018.
- **REST statt WebSocket**: Voxtral WebSocket ist laut Task-Spec nicht production-ready. REST `POST /v1/audio/transcriptions` ist die stabile Alternative.
- **`@capacitor/microphone` nicht installiert**: Paket existiert nicht auf npm. `getUserMedia` funktioniert nativ in Capacitor-Webviews und triggert den nativen Permission-Dialog automatisch.
