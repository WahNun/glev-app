# Task #166 — Drip-Mail-Fehler im Dashboard mit echtem Grund anzeigen

## Was wurde gemacht

Bisher leitete das Operator-Dashboard `/admin/drip` den "Fehlgeschlagen"-Status aus einer Zeit-Heuristik ab: jede Row, die mehr als 24h überfällig war und noch kein `sent_at` hatte, galt als failed. Der echte Resend-Fehlertext (z. B. "domain not verified") landete nur in den Server-Logs.

## Änderungen

**Migration `supabase/migrations/20260527_drip_error_tracking.sql`:**
- 3 neue Spalten in `email_drip_schedule`: `last_attempt_at` (timestamptz), `last_error` (text max 500 Zeichen), `attempt_count` (integer default 0)
- Partial Index auf `last_error IS NOT NULL AND sent_at IS NULL`

**`app/api/cron/drip/route.ts`:**
- Bei Resend-API-Fehler: UPDATE mit `last_error`, `last_attempt_at`, `attempt_count + 1`
- Bei unerwarteter Exception: dasselbe
- Bei Erfolg: `last_error = null` aufräumen + `last_attempt_at` setzen

**`lib/emails/drip-status.ts`:**
- `DripScheduleRow` Interface um die 3 neuen Felder erweitert
- `classifyRow()`: "failed" prüft jetzt `row.last_error != null` statt Zeit-Heuristik
- `dripBucketWindows()`: `failedThresholdIso` entfernt, `FAILED_GRACE_HOURS` entfernt

**`app/admin/drip/page.tsx`:**
- Counter-Query "failed": `.not("last_error","is",null)`
- due_today/tomorrow/week Counter: zusätzlich `.is("last_error",null)`
- Status-Filter "failed": `.not("last_error","is",null)`
- List-SELECT lädt die 3 neuen Felder mit

**`app/admin/drip/DripDashboard.tsx`:**
- Neue `ErrorCell`-Komponente: Fehlertext in Monospace-Rot (60 Zeichen + Tooltip mit vollständigem Text + ×N Versuche-Counter)
- Neue "Letzter Fehler"-Tabellenspalte mit ⓘ-Hint
- Bestätigungs-Modal zeigt Fehlertext wenn vorhanden

**`app/admin/drip/actions.ts`:**
- `sendNowAction`: bei Erfolg `last_error = null`; bei Fehler persistieren
- `rescheduleAction`: Kommentar dass Neu-Einplanen last_error nicht zurücksetzt

**`tests/unit/dripStatus.test.ts`:**
- Fixtures um 3 neue Felder erweitert
- Tests auf neue "failed via last_error"-Semantik umgestellt
- Überprüfung dass überfällige Row OHNE last_error = "due_today" bleibt
- `failedThresholdIso`-Test entfernt
