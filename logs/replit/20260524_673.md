# Task #673 — Draggable Meal-Node-Cluster auf der 12h-CGM-Curve + lokale BZ-Check-Reminder

## Summary
Auf der 12h-Glukosekurve im Dashboard (`CurrentDayGlucoseCard`) wird pro Mahlzeit-mit-Bolus in den letzten 12 h ein Make.com-Style Cluster gerendert: Zentrum-Node auf der Curve, ein draggable Pre-Knob und ein/mehrere Post-Knobs, plus „+"-Button für weitere `post_n`-Arme. Drag entlang X öffnet ein Confirm-Modal und schreibt nach Bestätigung in `meal_timeline_checks`. Zusätzlich wird per `@capacitor/local-notifications` ein lokaler Reminder mit eigenem Sound geplant.

## Spec
- Center-Node: 24 px, Knobs: 12 px, „+"-Button: 16 px
- Pre-Default: −15 min, Range: −60..−1
- Post-Default: +90 min, Range: +1..+180
- Unconfirmed Stub-Arme = gestrichelter Outline
- Drag → Confirm-Dialog (de/en) → Upsert in `meal_timeline_checks`
- Confirm zusätzlich: lokaler Reminder mit eigenem Sound (`glev_pre_check.wav` / `glev_post_check.wav`)

## Changes

### New files
- **`lib/mealTimelineChecks.ts`** — `listChecksForMeals(mealIds)` und `upsertCheck({mealId, checkType, plannedAt})`. Upsert läuft als Select-then-Update-or-Insert, weil die Tabelle (bislang nur als Phase-3-AI-Anker im Schema, ungenutzt) keinen Unique-Constraint auf `(meal_id, check_type)` hat. `confirmed_at = NOW()` wird beim Schreiben gesetzt.
- **`lib/mealCheckReminders.ts`** — `scheduleCheckReminder({mealId, checkType, plannedAt, title, body})`. Native (Capacitor `LocalNotifications`) mit custom Sound; Web-Fallback via `Notification`-API + `setTimeout`; SSR/Node no-op. Reminder-ID via FNV-1a Hash von `mealId|checkType` → erneuter Drag überschreibt denselben Slot statt zu stacken. Sound: `glev_pre_check.wav` für pre, `glev_post_check.wav` für post.
- **`components/MealNodeCluster.tsx`** — SVG-Cluster mit Drag (native PointerEvents + `setPointerCapture`), Haptics-Feedback, dashed-Outline für unbestätigte Arme, live Offset-Label. Confirm-Dialog via `createPortal` an `document.body` (statt `foreignObject` im SVG). Exportiert die reinen Math-Helpers `clampOffsetMin`, `pxDeltaToOffsetMin`, `kindOf`, `nextPostCheckType`, `defaultOffsetForNewPost` plus die Layout-Konstanten `CLUSTER_OVERLAP_PX=60` / `CLUSTER_STAGGER_Y_PX=14`.
- **`tests/unit/mealNodeMath.test.ts`** — 22 Playwright-Unit-Tests für alle Math-Helpers und die Default-/Range-Konstanten.

### Modified files
- **`components/CurrentDayGlucoseCard.tsx`** — `RollingChart` lädt parallel `fetchMeals({sinceDays:1, limit:50})` + `fetchRecentInsulinLogs(1)` und filtert auf das 12h-Fenster. Bolus-Filter: `insulin_logs.insulin_type === "bolus" && related_entry_id ∈ mealIds` ODER `meal.insulin_units > 0`. Cluster-Layout: Nearest-CGM-Y-Interpolation für die Zentrum-Y, plus Y-Stagger bei `|Δx| < CLUSTER_OVERLAP_PX`. Render in einem zweiten pointer-events-none `<svg>` Overlay, damit die interaktiven Knobs/„+"-Buttons re-opt-in können, ohne den Base-Chart anzufassen. `handleConfirmCheck` ruft `upsertCheck` + `scheduleCheckReminder` (fire-and-forget) und bumpt `reloadTick` für den dashed→solid Flip.
- **`messages/de.json` + `messages/en.json`** — neue Section `meal_timeline` mit 8 Keys: `pre_label`, `post_label`, `confirm_title`, `confirm_subtitle`, `confirm_button`, `cancel_button`, `offset_minutes_before`, `offset_minutes_after`, `add_post_arm_title`.
- **`DECISIONS.md`** — neuer **D-017** mit den Hard-Rules (Writes nur via explizitem UI-Confirm, kein Auto-Snap, kein Server-Push in dieser Iteration) + explizite Nicht-Entscheidungen aus der Task-Spec (`bg_at_check`, Landscape-Overlay, DB-Unique-Constraint). Fix-Log-Eintrag.
- **`replit.md`** — kurzer Bullet im System-Architecture-Abschnitt zum Meal-Node-Cluster mit Verweis auf D-017.
