# Task #174 — Show admins which exact day a drip-stats spike happened

**Date:** 2026-05-28
**Agent:** Replit Agent

## Summary

Added click/tap interactivity to the drip-stats admin sparkline so operators can see the exact day and counts behind any spike without needing to hover. The worst opt-out day is auto-highlighted on first paint, which works on touch devices.

## Changes

### New: `app/admin/drip-stats/DailySparkClient.tsx`
- `"use client"` React component replacing the static server-rendered `DailySpark` SVG
- `useState(defaultIdx)` — default is the bar with the highest `unsubscribed` count (computed via `useMemo`), so the worst day is highlighted immediately on load
- Clicking/tapping any bar selects it → renders a plain-text detail panel below: "14. Apr 2026 (Di): 12 Versendet, 8 Abgemeldet"
- Clicking the selected bar again deselects (toggles to `null`), panel shows a hint
- Full-height transparent `<rect>` hit-area per slot makes narrow bars easy to tap on mobile
- Selected column gets a faint red background; unsub bar darkens to confirm selection
- Accessible: `role="button"`, `tabIndex`, `aria-pressed`, `onKeyDown` (Enter/Space)

### Updated: `app/admin/drip-stats/page.tsx`
- Replaced static `DailySpark` function (~90 lines) with `<DailySparkClient>`
- Removed unused `DailyBucket` type import and `SPARK_SENT_COLOR` / `SPARK_UNSUB_COLOR` constants
- `SparkLegend` uses the same hex literals inline

### No changes to: `tests/unit/dripStats.test.ts`
- Existing 13 tests cover all pure-function logic (`aggregateDripStats`, `aggregateDailyDripSeries`, `formatRate`, reconciliation invariant) — no new test content needed for this pure UI change
